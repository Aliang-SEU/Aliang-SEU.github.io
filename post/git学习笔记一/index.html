Git简介

Git是一种版本管理工具，与SVN不同，它是一种分布式的版本控制系统，也就是说版本库在每一台主机上都会完整的保留一份下来，因此每个人可以离线进行工作，之后只需要将各自修改的内容推送给其他人即可进行协作。
那么Git是如何做到让互相之间都能够知道哪些文件被修改了，以及修改了哪些内容的呢，此外，具体的版本控制是如何进行的呢？还有如果多个人修改的内容互相冲突了之后又该怎么办？一切的一切都需要从Git的实现原理来进行解释。
